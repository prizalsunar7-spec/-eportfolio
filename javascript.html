JavaScript —
Interactivity & Logic

JavaScript (JS) is the only programming language that runs natively in web browsers. It enables dynamic behaviour: responding to user events, updating the DOM without page reloads, fetching data from APIs, validating forms, and running complex application logic. ES6+ (2015 and beyond) brought classes, arrow functions, destructuring, promises, modules, and async/await — transforming JS into a modern, powerful language.

Demonstration 1: Working Calculator

A fully functional calculator built with vanilla JavaScript. Demonstrates state management via module-level variables, event-driven architecture using inline onclick handlers, and a computed operations object to avoid verbose switch statements.

0
// Core calculator state & logic let current = '', previous = '', operator = ''; function calcFn(type, val) { if (type === 'num') { current += val; } else if (type === 'op') { previous = current; current = ''; operator = val; } else if (type === 'equals') { const a = parseFloat(previous), b = parseFloat(current); const ops = { '+': a+b, '-': a-b, '*': a*b, '/': a/b }; current = String(ops[operator]); } updateDisplay(); }
Key concepts: The calculator uses state variables (current, previous, operator), an event-driven architecture (onclick attributes call calcFn), and a computed object (ops) to avoid a verbose switch statement. DOM updates happen in a single updateDisplay() function — a unidirectional data-flow pattern mirrored by React and Vue.

Demonstration 2: Dynamic To-Do List

Add, complete, and delete tasks. Demonstrates array state management, DOM reconstruction via innerHTML, template literals, and event delegation.

    // JavaScript: add & render tasks let tasks = []; function addTodo() { const text = document.getElementById('todo-input').value.trim(); if (!text) return; tasks.push({ id: Date.now(), text, done: false }); renderTodos(); } function renderTodos() { const list = document.getElementById('todo-list'); list.innerHTML = tasks.map(t => ` <li class="${t.done ? 'done' : ''}"> <input type="checkbox" ${t.done ? 'checked' : ''} onchange="toggleTodo(${t.id})"> <span>${t.text}</span> <button onclick="removeTodo(${t.id})">✕</button> </li>`).join(''); }
    Key concepts: Tasks are stored in a JavaScript array of objects. Every state change (add, toggle, remove) mutates the array then calls renderTodos(), which rebuilds the list HTML using Array.map() and a template literal. This one-directional data flow mirrors the pattern used in React and Vue. escHtml() sanitises text to prevent XSS injection.

    Core JS Concepts

    JavaScript's power comes from a handful of foundational concepts that every developer needs to understand. Here are the most important ones with concise explanations.

    javascript fundamentals overview
    Closures

    A function that retains access to variables in its outer scope even after the outer function has returned. Enables private state and factory functions.

    Event Loop

    JavaScript is single-threaded. The event loop processes the call stack and task queue, enabling async behaviour via callbacks, promises, and async/await.

    Prototypal Inheritance

    Objects inherit from other objects via the prototype chain. ES6 class syntax is syntactic sugar over this mechanism.

    Destructuring

    Extract values from arrays or objects into variables in a single line: const {name, age} = user. Cleaner than multiple property accesses.

    Spread & Rest

    ...arr spreads items into a new array/call; ...rest collects remaining params. Both enable immutable patterns and variadic functions.

    Promises / Async

    Asynchronous operations (fetch, timers) return Promises. await pauses a function until the Promise resolves, keeping code readable without callback nesting.

    Array Methods

    map, filter, reduce, find, some, every — functional array transformations that avoid manual loops and mutation.

    Modules (ESM)

    export / import split code into reusable files. Tree-shaking removes unused exports at build time, reducing bundle size.
