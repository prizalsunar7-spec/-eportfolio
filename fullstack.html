What is
Full Stack Development?

Full stack development refers to the practice of working across the entire technology stack of a web application — both the front end (what users see and interact with) and the back end (servers, databases, and application logic). A full stack developer is proficient in both layers, enabling them to design, build, and deploy complete web applications independently.

🖥️
Front End

Client Side

Everything the user sees: HTML structures the content, CSS styles it, and JavaScript adds interactivity. Frameworks like React or Vue accelerate development. The browser renders this layer directly — no server required at runtime.

HTMLCSSJavaScriptReact
⚙️
Back End

Server Side

Business logic, authentication, and API design live here. The server receives HTTP requests, processes them, queries the database if needed, and returns structured responses (usually JSON). Common runtimes include Node.js, Python, and Java.

Node.jsPythonAPIsREST
🗄️
Database

Data Layer

Persistent storage for application data. Relational databases (MySQL, PostgreSQL) use tables and foreign keys; NoSQL databases (MongoDB) store flexible JSON documents. The choice depends on the data's shape and query patterns.

MySQLMongoDBPostgreSQL
☁️
DevOps

Deployment

Hosting, version control, and CI/CD pipelines ensure the application is available, scalable, and maintainable. Containerisation with Docker isolates environments. Cloud platforms (AWS, GCP, Azure) provide on-demand infrastructure.

GitDockerAWSCI/CD
Why Full Stack? A full stack developer can trace a bug from a button click on screen all the way to a database query — and fix it at any layer. This end-to-end understanding makes them invaluable for start-ups, agile teams, and solo projects alike. The most popular full stack combinations today are MERN (MongoDB, Express, React, Node) and LAMP (Linux, Apache, MySQL, PHP).

This ePortfolio itself demonstrates full stack awareness: the HTML provides semantic document structure, CSS and Bootstrap handle layout and design, JavaScript delivers client-side interactivity, and together they form a complete, deployable web page — the front-end layer of any full stack application.

Deep Dive

Popular Stacks

A "stack" is simply a named collection of technologies that work together across layers. Different stacks suit different use-cases — here are the most widely adopted ones.

popular full-stack combinations

MERN Stack

The modern JavaScript-everywhere approach. All four layers use JS or JSON, so developers need only one language across the entire application.

// MERN = MongoDB + Express + React + Node.js MongoDB → document database (JSON-like) Express → minimal Node.js web framework React → declarative front-end UI library Node.js → JavaScript runtime for the server
JavaScriptJSONNoSQL

LAMP Stack

The classic open-source combination powering millions of websites including early WordPress and Wikipedia. Battle-tested and well-documented.

// LAMP = Linux + Apache + MySQL + PHP Linux → operating system (Ubuntu, CentOS) Apache → HTTP web server MySQL → relational database PHP → server-side scripting language
Open SourceRelationalMature

Django + React

Python's batteries-included framework paired with the most popular front-end library. Common in data science and fintech contexts.

PythonREST APIReact

JAMstack

JavaScript, APIs, and Markup — pre-rendered static sites that call third-party APIs at runtime. Extremely fast and secure, popular with Next.js and Netlify.

StaticCDNNext.js
Roadmap

Learning Roadmap

Becoming a full stack developer is a journey of layered skill-building. Each layer depends on the one beneath it — master the foundations before adding complexity.

recommended learning path
1

HTML & CSS Fundamentals

Learn semantic markup, the box model, Flexbox, Grid, and responsive design. Everything else builds on this foundation. Time estimate: 4–6 weeks.

HTML5CSS3Responsive
2

JavaScript (Vanilla)

Variables, functions, arrays, objects, DOM manipulation, event listeners, fetch API, and ES6+ syntax. Build interactive projects — a calculator, a quiz app, a weather widget. Time estimate: 6–10 weeks.

ES6+DOMAsync/Await
3

A Front-End Framework

React is the industry standard. Learn components, props, state (useState), effects (useEffect), routing (React Router), and state management (Context or Redux Toolkit). Time estimate: 8–12 weeks.

ReactComponentsHooks
4

Back-End & APIs

Node.js with Express or Python with Django/FastAPI. Learn HTTP, REST API design, middleware, authentication (JWT, sessions), and error handling. Time estimate: 6–8 weeks.

Node.jsRESTAuth
5

Databases & Deployment

Learn SQL basics and one relational DB (PostgreSQL) plus one NoSQL DB (MongoDB). Then deploy with Vercel/Netlify (front end) and Railway/Render (back end). Add version control with Git and GitHub from day one. Time estimate: ongoing.

PostgreSQLMongoDBGitDeploy
