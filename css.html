CSS —
Styling & Layout

Cascading Style Sheets (CSS) control the visual presentation of HTML. Modern CSS includes powerful layout tools (Flexbox, Grid), custom properties (variables), transitions, animations, and the cascade — the rules that determine which styles apply when multiple rules target the same element. CSS3, the current iteration, brought modular specifications that browsers implement incrementally.

/* CSS producing the spring hover effect */ .css-card { transition: transform 0.25s cubic-bezier(.34,1.56,.64,1), box-shadow 0.25s ease; } .css-card:hover { transform: translateY(-6px) scale(1.03); box-shadow: 0 16px 40px rgba(0,0,0,0.15); }
cubic-bezier(.34,1.56,.64,1) creates a spring-like overshoot — the card bounces slightly past its destination before settling. This micro-interaction communicates interactivity far better than a simple linear or ease transition, and adds life to a static layout.

Demonstration: The Box Model

Every HTML element is a rectangular box composed of four layers. Understanding the box model is essential for precise layout control. Setting box-sizing: border-box (a near-universal modern practice) makes width include padding and border, eliminating a common source of layout bugs.

.box { margin: 20px; /* space OUTSIDE the border */ border: 6px solid #4caf50; padding: 16px; /* space INSIDE the border */ width: 200px; /* width of the content area */ box-sizing: border-box; /* include padding+border in width */ }

Demonstration: Flexbox & CSS Grid

Flexbox makes one-dimensional (row or column) layouts trivial. The navigation bar at the top of this page uses Flexbox. Below is a live example demonstrating justify-content and align-items:

flex-start
center (taller)
flex-end
.container { display: flex; justify-content: space-between; /* horizontal (main) axis */ align-items: center; /* vertical (cross) axis */ gap: 0.5rem; flex-wrap: wrap; /* wrap on small screens */ }
Flexbox vs Grid: Use Flexbox for 1D layouts — nav bars, button groups, centering content. Use CSS Grid for 2D layouts — page-level structure, card grids, dashboard panels. They complement each other and are often nested.

CSS Animations

CSS animations use @keyframes to define intermediate states, and the animation property to apply them. Unlike transitions (which require a trigger like hover), animations run automatically and can loop.

demo · @keyframes animations

translateX

/* Pulse animation */ @keyframes pulse { 0%, 100% { transform: scale(1); } 50% { transform: scale(1.15); } } .element { animation: pulse 1.5s ease-in-out infinite; /* name duration easing iteration-count */ }
Performance tip: Only animate transform and opacity for GPU-accelerated, jank-free animations. Animating width, height, top, or margin triggers layout recalculation on every frame — expensive on complex pages. Use will-change: transform to hint the browser to promote the element to its own compositor layer.

Custom Properties & Specificity

CSS custom properties (variables) allow you to store values in reusable tokens. The cascade and specificity rules determine which declarations win when multiple rules conflict.

demo · css variables & the cascade

CSS Custom Properties (Variables)

var(--ink) background · var(--gold) text
border: var(--accent) · transparent bg
/* Define in :root — accessible everywhere */ :root { --ink: #0d0d0d; --paper: #f5f0e8; --accent: #c0392b; --gold: #d4a843; } /* Use with var() — theming in one place */ body { background: var(--paper); color: var(--ink); } a:hover { color: var(--gold); }

Specificity — Who Wins?

1-0-0
#id — ID selector — highest specificity
0-1-0
.class, [attr], :pseudo-class
0-0-1
element selector — lowest specificity
The cascade order (highest to lowest): !important → inline styles → specificity score → source order. Prefer class selectors over IDs for styles; keep specificity low and predictable to avoid specificity wars.
